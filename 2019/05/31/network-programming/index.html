Socket Address Structures

Sockets Interface

getaddrinfo

Convert string representations of hostnames, host addresses, ports, and service names to socket address structures.

Each addrinfo struct returned by getaddrinfo contains arguments that can be passed directly to socket function.

  • Return Value
    0 if it succeeds, or one of the several nonzero error codes

addrinfo structure

  • Client walk this list, trying each socket address in turn, until the calls to socket and connect succeed.
  • Servers walk the list until call to socket and bind succeed.

getnameinfo

Inverse of getaddrinfo, converting a socket address to the corresponding host and service.

  • Return Value
    On success, 0 is returned, and node and service names, if requested, are filled with null-terminated strings, possibly truncated to fit the specified buffer lengths.
    On error, one of the several nonzero error codes is returned

Example 1

socket

create a socket descriptor

AF_INET indicates that we are using 32-bit IPV4 addresses
SOCK_STREAM indicates that the socket will be the end point of a connection

  • Return Value
    On success, a file descriptor for the new socket is returned. On error, -1 is returned, and errno is set appropriately.

bind

A server uses bind() to ask the kernel to associate the server’s socket address with a socket descriptor.

  • Return Value
    On success, zero is returned. On error, -1 is returned, and errno is set appropriately.

listen

By default, kernel assumes that descriptor from socket function is an active socket that will be on the client end of a connection.

A server calls the listen function to tell the kernel that a descriptor will be used by a server rather than a client.

Converts sockfd it from an active socket to a listening socket that can accept connection requests from clients.

backlog is a hint about the number of outstanding conneciton requests that the kernel should queue up before starting to refuse requests. (listen backlog)

  • Return Value
    On success, zero is returned. On error, -1 is returned, and errno is
    set appropriately.

accept

Servers wait for connection requests from clients by calling accept()

Waits for connection request to arrive on the connection bound to listenfd, then fills in client’s socket address in addr and size of the socket address in addrlen.

  • Return Value
    On success, a connected descriptor (connfd) that can be used to communicate with the client via Unix I/O routines.

connect

A client establishes a connection with a server by calling this.

If successful, then clientfd is now ready for reading and writing.

  • Return Value
    If the connection or binding succeeds, zero is returned. On error, -1 is returned, and errno is set appropriately.
